Introducción¶

En este documento se pretende mostrar cmo crear modelos de machine learning combinando H2O y el lenguaje de programacin Python.

Aunque son muchos los pasos que preceden al entrenamiento de un modelo (exploracin de los datos, transformaciones, seleccin de predictores, etc.), para no aadir una capa extra de complejidad, se va a asumir que los datos se encuentran prcticamente preparados para que puedan ser aceptados por los algoritmos de aprendizaje. Se puede encontrar una descripcin ms detallada de las etapas que forman parte el ciclo de vida del modelado en el documento Machine Learning con R y caret.

¿Qué es H2O?¶

H2O es un producto creado por la compaa H2O.ai con el objetivo de combinar los principales algoritmos de machine learning y aprendizaje estadstico con el Big Data. Gracias a su forma de comprimir y almacenar los datos, H2O es capaz de trabajar con millones de registros en un nico ordenador (emplea todos sus cores) o en un cluster de muchos ordenadores. Internamente, H2O est escrito en Java y sigue el paradigma Key/Value para almacenar los datos y Map/Reduce para sus algoritmos. Gracias a sus API, es posible acceder a todas sus funciones desde R, Python o Scala, as como por una interfaz web llamada Flow.

Aunque la principal ventaja de H2O frente a otras herramientas es su escalabilidad, sus algoritmos son igualmente tiles cuando se trabaja con un volumen de datos reducido.

Comunicación entre H2O y Python¶

El manejo de H2O puede hacerse ntegramente desde Python: iniciar el cluster, carga de datos, entrenamiento de modelos, prediccin de nuevas observaciones, etc. Es importante tener en cuenta que, aunque los comandos se ejecuten desde Python, los datos se encuentran en el cluster de H2O no en memoria. Solo cuando los datos se cargan en memoria, se les pueden aplicar funciones propias de Python.

Las funciones h2o.H2OFrame() y as_data_frame(use_pandas=True) permiten transferir los datos de la sesin de Python al cluster H2O y viceversa. Hay que tener especial precaucin cuando el movimiento de datos se hace desde H2O a Python, ya que esto implica cargar en RAM todos los datos y, si son muchos, pueden ocupar toda la memoria. Para evitar este tipo de problemas, conviene realizar todos los pasos posibles (filtrado, agregaciones, clculo de nuevas columnas...) con las funciones de H2O antes de transferir los datos.

Datos¶

Para los ejemplos de este documento se emplea el set de datos Adult Data Set disponible en UCI Machine Learning Repository. Este set de datos contiene informacin sobre 48842 ciudadanos de diferentes pases. Para cada ciudadano, se dispone de 15 variables:

  • workclass: tipo de empleo.
  • fnlwgt: importancia relativa de cada observacin.
  • education: nivel de estudios.
  • education-num: aos de estudios.
  • marital-status: estado civil.
  • occupation: sector de trabajo

Separación de training, validación y test¶
















Evaluar la capacidad predictiva de un modelo consiste en comprobar cmo de prximas son sus predicciones a los verdaderos valores de la variable respuesta. Para poder cuantificar de forma correcta este error, se necesita disponer de un conjunto de observaciones, de las que se conozca la variable respuesta, pero que el modelo no haya "visto", es decir, que no hayan participado en su ajuste. Con esta finalidad, se separan los datos disponibles en un conjunto de entrenamiento, uno de validacin y uno de test. El conjunto de entrenamiento se emplea para ajustar el modelo, el de validacin para encontrar los mejores hiperparmetros (model tuning) y el de test para estimar el error que comete el modelo al exponerse a nuevos datos. El tamao adecuado de las particiones depende en gran medida de la cantidad de datos disponibles y la seguridad que se necesite en la estimacin del error, 70%-15%-15% suele dar buenos resultados.


Si el nmero de observaciones es limitado, crear 3 particiones puede generar grupos demasiado pequeos y variables. En estos casos, es preferible dividir los datos nicamente en un conjunto de entrenamiento y otro de test, y utilizar validacin cruzada (cross validation) sobre el conjunto de entrenamiento durante la optimizacin de los hiperparmetros del modelo. Para mostrar ambas posibilidades, en el ejemplo  de modelo GLM se emplear validacin cruzada, mientras que en el resto de modelos (GBM, RF y Deep Learning), se emplea un solo conjunto de validacin. Es importante tener en cuenta que, la validacin cruzada, requiere ajustar el modelo repetidas veces, lo que supone un coste computacional muy alto. Cuando se trabaja con big data no suele ser factible aplicar validacin cruzada, adems, al disponer de tantos datos, un conjunto de validacin nico es suficiente para obtener buenas estimaciones del comportamiento del modelo. Para una descripcin ms detallada de los diferentes mtodos de validacin, consultar el Anexo4 del documento Machine learning con R y Caret.


El mtodo .splitFrame() realiza particiones aleatorias, pero no permite hacerlas de forma estratificada, por lo que no asegura que la distribucin de clases de variable respuesta sea igual en todas particiones. Esto puede ser problemtico con datos muy desbalanceados (alguno de los grupos es muy minoritario). Para obtener un reparto estratificado puede emplearse el mtodo stratified_split().

Por defecto, H2O estandariza los predictores numricos antes de ajustar los modelos para que todos tengan media cero y varianza uno. Eliminación de variables con varianza cero¶
















No se deben de incluir en un modelo predictores que contengan un nico valor (varianza cero), ya que no aportan informacin. Los algoritmos de H2O excluyen directamente las columnas con valor constante. Este comportamiento se puede controlar mediante el argumento ignore_const_cols.

















Balance de clases¶
















En problemas de clasificacin, puede ocurrir que los datos estn desbalanceados, es decir, que el nmero de observaciones pertenecientes a cada grupo sea muy dispar. La forma ms comn de encontrar los valores ptimos es probando diferentes posibilidades, lo que comnmente se conoce como grid search.

Generalized linear models (GLMs)¶
















Introducción¶
















El modelo lineal generalizado surge como resultado de generalizar y unificar diferentes modelos lineales tradicionales (regresin lineal por mnimos cuadrados, regresin logstica...), bajo un mismo marco.


En el modelo de regresin lineal por mnimos cuadrados, la variable respuesta y est linealmente relacionada con el o los predictores X acorde a la ecuacin:


$$y = \beta_0 + X^T \beta + \epsilon $$


Para que esta aproximacin sea vlida, se necesita que el error se distribuya de forma normal y que la varianza sea constante. Estos requisitos hacen que el modelo no se aplicable a muchos problemas reales. EL GLM permite solucionar esta limitacin haciendo una abstraccin a un modelo ms genrico en el que la distribucin del error puede seguir distintas funciones link (Gaussian, Poisson, binomial...). Por ejemplo, el modelo lineal generalizado equivale al modelo lineal por mnimos cuadrados cuando la funcin link es gausiana, o a la regresin logstica cuando es binomial (logit). Adems, el modelo lineal generalizado puede incluir regularizacin durante su ajuste, por lo que tambin incluye los modelos ridge regression, lasso y elastic net.


El ajuste de todos los modelos GLM consiste en identificar el valor de los coeficientes $\beta_0$ y $\beta$ que maximizan la verosimilitud (likelihood) de los datos, es decir, los valores de los coeficientes que dan lugar al modelo que con mayor probabilidad puede haber generado los datos observados. Cuando se emplea una funcin link gausiana (con o sin regularizacin L2), la mxima verosimilitud equivale a minimizar la suma de errores cuadrados, lo que tiene una solucin analtica. Para el resto de familias o cuando se emplea la regularizacin L1, no existe una solucin analtica con la que encontrar la mxima verosimilitud, por lo que se requiere un mtodo de optimizacin iterativo como el iteratively reweighted least squares (IRLSM), L-BFGS, mtodo de Newton o descenso de gradiente. H2O selecciona automticamente el mtodo apropiado en dependiendo de la funcin link especificada por el usuario.

















Regularización¶
















H2O incorpora 3 tipos de regularizacin para los modelos GLM con el objetivo de evitar overfitting, reducir varianza y atenuar el efecto de la correlacin entre predictores. Por lo general, aplicando regularizacin se consigue modelos con mayor poder predictivo (generalizacin).


    

  • El modelo lasso es un modelo lineal por mnimos cuadrados que incorpora una regularizacin que penaliza la suma del valor absolutos de los coeficientes de regresin $(||\beta||_1 = \sum_{k=1} ^p |\beta_k|)$. Ha esta penalizacin se le conoce como l1 y tiene el efecto de forzar a que los coeficientes de los predictores tiendan a cero. Dado que un predictor con coeficiente de regresin cero no influye en el modelo, lasso consigue seleccionar los predictores ms influyentes. El grado de penalizacin est controlado por el hiperparmetro $\lambda$. Cuando $\lambda = 0$, el resultado es equivalente al de un modelo lineal por mnimos cuadrados ordinarios. A medida que $\lambda$ aumenta, mayor es la penalizacin y ms predictores quedan excluidos.

    

    • 

  • El modelo ridge es un modelo lineal por mnimos cuadrados que incorpora una regularizacin que penaliza la suma de los coeficientes elevados al cuadrado $(||\beta||^2_2 = \sum_{k=1} ^p \beta^2_k)$. Ha esta penalizacin se le conoce como l2 y tiene el efecto de reducir de forma proporcional el valor de todos los coeficientes del modelo pero sin que estos lleguen a cero. Al igual que lasso, el grado de penalizacin est controlado por el hiperparmetro $\lambda$.
    

    • 



La principal diferencia prctica entre lasso y ridge es que el primero consigue que algunos coeficientes sean exactamente cero, por lo que realiza seleccin de predictores, mientras que el segundo no llega a excluir ninguno. Esto supone una ventaja notable de lasso en escenarios donde no todos los predictores son importantes para el modelo y se desea que los menos influyentes queden excluidos. Por otro lado, cuando existen predictores altamente correlacionados (linealmente), ridge reduce la influencia de todos ellos a la vez y de forma proporcional, mientras que lasso tiende a seleccionar uno de ellos, dndole todo el peso y excluyendo al resto. En presencia de correlaciones, esta seleccin vara mucho con pequeas perturbaciones (cambios en los datos de entrenamiento), por lo que, las soluciones de lasso, son muy inestables si los predictores estn altamente correlacionados.

















Para conseguir un equilibrio ptimo entre estas dos propiedades, se puede emplear lo que se conoce como penalizacin elastic net, que combina ambas estrategias.


    

  • El modelo elastic net incluye una regularizacin que combina la penalizacin l1 y l2 $(\alpha \lambda ||\beta||_1 + \frac{1}{2}(1- \alpha)||\beta||^2_2)$. El grado en que influye cada una de las penalizaciones est controlado por el hiperparmetro $\alpha$. Su valor debe estar comprendido en el intervalo [0,1], cuando $\alpha = 0$, H2O aplica ridge regression y cuando $\alpha = 1$ aplica lasso. La combinacin de ambas penalizaciones suele dar lugar a buenos resultados. Una estrategia frecuentemente utilizada es asignarle casi todo el peso a la penalizacin l1 ($\alpha$ muy prximo a 1) para conseguir seleccionar predictores y un poco a la l2 para dar cierta estabilidad en el caso de que algunos predictores estn correlacionados.
    • 



Encontrar el mejor modelo GLM implica identificar los valores ptimos de los hiperparmetros de regularizacin $\alpha$ y $\lambda$. Por defecto, H2O selecciona $\alpha = 0.5$ y, con cada uno de ellos, analiza un rango de valores $\lambda$. Tambin es posible  hacer una bsqueda del valor ptimo de $\alpha$ mediante un grid seach.

















Lambda Search



La bsqueda del valor ptimo de lambda se habilita indicando lambda_search = True. Adems, esta bsqueda puede ser configurada con los siguientes argumentos:


    

  • alpha: el valor de $\alpha$ que se emplea en el modelo para distribuir la penalizacin l1 y l2. Por defecto, alpha=0.5.

    

    • 

  • validation_frame: mtodo de validacin empleado para identificar el mejor modelo, puede ser mediante un nico conjunto de validacin o mediante validacin cruzada (cross-validation). En el segundo caso, no es necesario crear un conjunto de validacin, las particiones se generan a partir del conjunto de entrenamiento.

    

    • 

  • n_folds: nmero de particiones en la validacin cruzada.

    

    • 

  • lambda_min_ratio y nlambdas: determinan la secuencia de valores lambda que se generan automticamente y sobre los que se realiza la bsqueda. El rango de valores va desde el valor mnimo con el que se alcanza una penalizacin total (lambda_max), es decir, el valor de lambda a partir del cual el coeficiente de todos los predictores es cero (valor identificado automticamente por H2O), hasta un valor de lambda igual a lambda_min_ratio x lambda_max. Dentro de ese rango, se generan un nmero de lambdas igual al valor especificado con nlambdas. Por defecto, se emplean nlambda=100 y  lambda_min_ratio=1e4.

    

    • 

  • max_active_predictors: establece el nmero mximo de predictores incluidos en el modelo (predictores con coeficientes de regresin distintos de cero). Esta opcin es muy til para evitar tiempos de computacin excesivos cuando se trabaja con sets de datos que contienen cientos de predictores pero se sabe que solo unos pocos son importantes.

    

    • 


















Método de ajuste¶
















Todos los modelos GLM se ajustan encontrando el valor de los coeficientes $\beta$ que maximizan el likelihood. La estimacin de estos coeficientes se realiza mediante mtodos iterativos, ya que, a excepcin de unos pocos casos, no existe una solucin analtica. H2O incorpora dos mtodos (solvers) distintos, cuya correcta eleccin, que depende del tipo de datos, puede influir notablemente en el rendimiento computacional.


    

  • Iteratively Reweighted Least Squares Method (IRLSM): funciona especialmente bien cuando el set de datos empleado tiene muchas observaciones (largo) pero un nmero limitado de predictores (estrecho). Los autores recomiendan utilizarlo cuando el nmero de predictores no es mayor de 500.

    

    • 

  • Limited-memory Broyden-Fletcher-Goldfarb-Shanno algorithm (L-BFGS): es el mtodo recomendado cuando se dispone de muchos predictores (>500).
    

    • 



Las anteriores son solo recomendaciones generales, en caso de duda, la mejor forma de elegir es comparando el resultado de ambos solvers.

En el caso de los GLM, el criterio de parada es el nmero mximo de predictores permitidos, que se especifica mediante el argumento max_active_predictors.

Coeficientes de regresión¶
















Si se especifica en la creacin del modelo que se estandaricen los predictores (standardize=True), se generan dos tipos de coeficientes de regresin:


    

  • Coeficientes estandarizados: son los obtenidos directamente por el modelo al emplear los predictores estandarizados. Es importante tener en cuenta que, al estar estandarizados, su valor no est en las mismas unidades que los predictores originales, lo que dificulta su interpretacin. Sin embargo, s que son tiles (su valor absoluto) para ordenar los predictores de mayor a menor influencia en el modelo.

    

    • 

  • Coeficientes: son los coeficientes "normales" obtenidos a partir de los coeficientes estandarizados pero revirtiendo la estandarizacin. Sus unidades son las mismas que las de los predictores originales y pueden ser interpretados de forma normal.
    

    • 



Para obtener ms informacin sobre la correcta interpretacin de los coeficientes de regresin de un modelo de regresin logstica consultar Regresin logstica simple y mltiple en R.

Gradient Boosting Machine (GBM)¶
















Introducción¶
















Boosting Machine (BM) es un tipo de modelo obtenido al combinar (ensembling) mltiples modelos sencillos (de regresin o clasificacin), tambin conocidos como weak learners. Est combinacin se realiza de forma secuencial, de forma que cada nuevo modelo que se incorpora al conjunto intenta corregir los errores de los anteriores modelos. Como resultado de la combinacin de mltiples modelos, Boosting Machine consigue aprender relaciones no lineales entre la variable respuesta y los predictores. Si bien los modelos combinados pueden ser muy variados, H2O, al igual que la mayora de libreras, utiliza como weak learners modelos basados en rboles.


Gradient Boosting Machine (GBM) es una generalizacin del modelo de Boosting Machine que permite aplicar el mtodo de descenso de gradiente para optimizar cualquier funcin de coste durante el ajuste del modelo.


El valor predicho por un modelo GBM es la agregacin (normalmente la moda en problemas de clasificacin y la media en problemas de regresin) de las predicciones de todos los modelos individuales que forman el ensemble.

















Dado que el ajuste de los weak learners que forman un GBM se hace de forma secuencial (el rbol i se ajusta a partir de los resultados del rbol i-1), las posibilidades de paralelizar el algoritmo son limitadas. H2O consigue acelerar el proceso en cierta medida paralelizando el ajuste de los rboles individuales.


Para informacin ms detallada sobre mtodos boosting consultar rboles de prediccin.

















Detención temprana del entrenamiento (Early Stopping)¶
















Una de las caractersticas de GBM es que, con el nmero suficiente de weak learners, el modelo final tiende a ajustarse perfectamente a los datos de entrenamiento causando overfitting. Este comportamiento implica que el analista tiene que encontrar el nmero adecuado de rboles y, para ello, suele tener que entrenar el modelo con cientos o miles de rboles hasta identificar el momento en el que empieza el overfitting. Esto suele ser poco eficiente en trminos de tiempo, ya que, posiblemente, se estn ajustando muchos rboles innecesarios.


De nuevo, la experiencia de los creadores de H2O queda reflejada en su herramienta, esta vez incluyendo toda una serie de estrategias para detener el proceso de ajuste de un modelo GBM a partir del momento en el que este deja de mejorar. Por defecto, la mtrica empleada se calcula con los datos de validacin. Cuatro argumentos controlan la estrategia de parada:


    

  • stopping_metric: mtrica empleada para cuantificar cunto mejora el modelo.
    

    • 

  • score_tree_interval: nmero de rboles tras los que se evala el modelo. Por ejemplo, si el valor es 10, entonces, tras cada 10 nuevos rboles que se aaden al modelo, se calcula la stopping_metric. Si bien la evaluacin del modelo puede hacerse tras cada nuevo rbol que se aade, esto puede ralentizar el entrenamiento.
    

    • 

  • stopping_tolerance: porcentaje mnimo de mejora entre dos mediciones consecutivas (score_tree_interval) por debajo del cual se considera que el modelo no ha mejorado.
    

    • 

  • stopping_rounds: nmero de mediciones consecutivas en las que no se debe superar el stopping_tolerance para que el algoritmo se detenga.
    

    • 



Algunos ejemplos de criterios de parada son:


    

  • Detener el entrenamiento de un modelo cuando el error de clasificacin no se reduce ms de un 1% durante dos mediciones consecutivas: stopping_rounds=1, stopping_tolerance=0.01 y stopping_metric="misclassification".
    

    • 

  • Detener el entrenamiento de un modelo cuando el logloss no se reduce nada durante 3 mediciones consecutivas: stopping_rounds=3, stopping_tolerance=0 y stopping_metric="logloss".
    

    • 

  • Detener el entrenamiento de un modelo cuando la media de AUC no aumenta ms de un 0.1% durante cinco mediciones consecutivas: stopping_rounds=5, stopping_tolerance=0.001 y stopping_metric="AUC".
    

    • 

  • Para detener el modelo aunque la mtrica converja, se tiene que deshabilitar este comportamiento de parada automtica indicando stopping_rounds=0. En este caso, el algoritmo se detendr cuando se alcance otra condicin de parada, por ejemplo, el nmero mximo de rboles que pueden formar el modelo.
    

    • 



Tambin es posible detener el entrenamiento del modelo cuando se supera un tiempo mximo especificado con el argumento max_runtimesecs > 0.


En la prctica, es conveniente establecer un nmero mximo de rboles muy elevado (potencialmente por encima de la cantidad necesaria) y activar la parada temprana.

















Comportamiento estocástico¶
















Stochastic Gradient Boosting es el resultado de emplear, en el entrenamiento de cada rbol, nicamente una seleccin aleatoria de observaciones y predictores del conjunto de entrenamiento. Esta modificacin suele reducir el overfitting del modelo y mejorar su generalizacin. Se puede especificar el % de datos aleatorios seleccionados en cada ajuste con los argumentos:


    

  • sample_rate: porcentaje de observaciones aleatorias empleadas para el ajuste de cada rbol del ensemble.
    

    • 

  • sample_rate_per_class: cuando el set de datos con el que se trabaja est desbalanceado (el % de observaciones de cada grupo es muy distinto), al hacer una seleccin aleatoria de observaciones para el entrenamiento de los rboles, puede ocurrir que, en algunos rboles, la seleccin excluya a alguno de los grupos. Para evitarlo, con el argumento sample_rate_per_class se indica que la seleccin debe hacerse de forma estratificada, asegurando que todos los grupos queden representados.
    

    • 

  • col_sample_rate: porcentaje de predictores (columnas) seleccionados de forma aleatoria para el ajuste de cada rbol del ensemble.
    

    • 



Los tres parmetros deben tener un valor entre 0 y 1. Por defecto, el valor empleado es 1, es decir, se emplean todos los datos y no hay comportamiento estocstico.

















Distribución y función de coste¶
















Dependiendo el tipo de modelo (regresin, clasificacin binaria, clasificacin mltiple...) se emplea una distribucin y una funcin de coste distinta. Indicando la distribucin con el argumento distribution, H2O selecciona automticamente la funcin de coste correspondiente.


Las distribuciones disponibles son: bernoulli, quasibinomial, multinomial, poisson, laplace, tweedie, gaussian, huber, gamma y quantile. De entre todas ellas, las ms empleadas suelen ser: gaussian para regresin, bernoulli para clasificacin binaria y multinomial para clasificacin multiclase. Aunque menos frecuentes, tambin pueden resultar muy tiles la distribucin huber, cuando se quiere que los errores penalicen ms que su valor absoluto pero menos que el cuadrado, y la distribucin quantile cuando en lugar de la media se quiere predecir uno de los cuantiles, por ejemplo la mediana.

















Otros hiperparámetros¶
















Adems de los ya descritos (criterio de parada, comportamiento estocstico, distribucin), el modelo GBM de H2O incorpora muchos otros parmetros e hiperparmetros con los que se puede controlar el comportamiento del algoritmo. A continuacin se describen algunos de los que tienen ms influencia en el modelo final. Se puede encontrar un listado completo en la documentacin.


Complejidad de los rboles



    

  • ntrees: nmero de rboles que forman el ensemble.
    

    • 

  • max_depth: profundidad mxima que pueden tener los rboles (nmero mximo de nodos).
    

    • 

  • min_rows: nmero mnimo de observaciones que debe tener cada nodo.
    

    • 

  • min_split_improvement: reduccin mnima en el error cuadrtico que debe de conseguir la divisin de un nodo para que se lleve a cabo.


    

    • 



Velocidad de aprendizaje



    

  • learn_rate: determina cuanto influye cada rbol individual en el conjunto del ensemble. Cuanto ms pequeo es el valor de learning rate, ms lentamente aprende el modelo y ms rboles se necesitan, sin embargo, al aprender poco a poco, se reduce el riesgo de overfitting. Suelen emplearse valores inferiores a 0.1.
    

    • 

  • learn_rate_annealing: emplear un valor de learn_rate muy pequeo puede conllevar que el modelo necesite varios miles de rboles y, por lo tanto, el entrenamiento tarde mucho tiempo. En lugar de utilizar un valor muy bajo desde el principio, se puede iniciar el proceso con un valor moderadamente alto e ir reducindolo tras cada nuevo rbol que se incorpora al modelo.  El learn_rate_annealing determina el factor por el que se reduce el learn_rate en cada iteracin. Para N rboles, GBM empieza con learn_rate y termina con learn_rate * learn_rate_annealing^N. Por ejemplo, en lugar de emplear earn_rate=0.01, se puede emplear un learn_rate=0.05 y un learn_rate_annealing=0.99. Con la segunda opcin, el algoritmo converge mucho ms rpido con aproximadamente la misma capacidad predictiva final.


    

    • 



Comportamiento monotnico



Los modelos GBM son capaces de aprender interacciones complejas entre los predictores y la variable respuesta. Esto puede convertirse en una desventaja cuando, debido al ruido de los datos, el modelo aprende comportamientos que no reflejan la realidad. Con el argumento monotone_constraints se puede forzar a que la relacin de cada predictor (numrico) con la variable respuesta tenga siempre la misma direccin (positiva o negativa). Puede encontrarse una descripcin ms detallada en GBM monotnicos.




Para conseguir maximizar la capacidad predictiva de un modelo GBM, es imprescindible entender bien cmo influye cada uno de los parmetros e hiperparmetros. Algunos de ellos pueden encontrarse descritos con ms detalle en el documento Boosting.

Grid search¶
















Empleando los valores por defecto de los hiperparmetros, se ha conseguido un AUC de test de 0.893. A continuacin, se intenta encontrar los valores ptimos de los hiperparmetros con los que se consiga mejorar an ms el AUC.


Dada la gran cantidad de hiperparmetros que tiene el algoritmo GBM y, aunque todos interaccionan entre s, no es posible explorar todo el espacio de posibles valores a la vez. Una estrategia que suele funcionar bien es establecer primero los hiperparmetros ms influyentes (nmero de rboles, learning rate, complejidad de los rboles, % de observaciones empleadas en cada ajuste).


    

  • Nmero de rboles (ntree): incorporar tantos rboles como sea necesario hasta que el error de validacin se estabilice o empiece a aumentar. Dado que H2O incorpora estrategias de parada temprana, no hay problema en indicar un nmero elevado porque, de no ser necesario, no se ajustarn.
    

    • 

  • Tasa de aprendizaje (learning_rate): por lo general, valores pequeos (<0.01) dan lugar a en mejores modelos, pero requieren ms rboles para aprender. Se recomienda reducir el valor tanto como permitan las limitaciones de tiempo/computacin.
    

    • 

  • Profundidad de los rboles (max_depth): es la principal forma de controlar la complejidad de los rboles que forman el ensemble. Al tratarse de weak learners, la complejidad de los rboles debe ser baja, aunque depende mucho de los datos, el valor ptimo raramente est fuera del rango [1, 20].
    

    • 

  • Comportamiento estocstico (sample_rate, col_sample_rate): la finalidad de emplear solo un subconjunto de observaciones y/o columnas en el entrenamiento de cada rbol es conseguir mejor generalizacin, sin embargo, puede tener el resultado contrario cuando se dispone de muy pocos datos. El valor ptimo suele estar dentro del rango de [0.7-0.9]. En situaciones en las que la variable respuesta es cualitativa y los grupos estn desbalanceados, es conveniente realizar una seleccin aleatoria estratificada, este comportamiento puede activarse con sample_rate_per_class.
    

    •
